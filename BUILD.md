# Build & Launch Guide

## Prerequisites

**Compiler & Build Tools:**
- MinGW64 (gcc, make) installed at `C:/msys64/mingw64/`
- CMake installed and in PATH

**Runtime:**
- SDL2.dll in `build/` directory (copied from MinGW after build)

## Build Directory Structure

```
build/
├── CMakeLists.txt        (symlinked or copied from root)
├── CMakeFiles/           (generated by cmake - safe to delete)
├── Makefile              (generated by cmake - safe to delete)
├── pokered.exe           (built binary)
├── SDL2.dll              (MUST COPY - see below)
└── *.obj files           (compiled objects)
```

**Safe to delete:** `CMakeFiles/`, `Makefile`, `*.obj` files  
**DO NOT delete:** `build/` directory itself, or you lose CMake configuration

## Build Process

### 1. Initial Setup (One-time)

```bash
cd C:/Users/Anthony/pokered
mkdir -p build
cd build

# Configure with MinGW explicitly
/c/msys64/mingw64/bin/cmake.exe -G "MinGW Makefiles" \
  -DCMAKE_C_COMPILER="C:/msys64/mingw64/bin/gcc.exe" \
  -DCMAKE_MAKE_PROGRAM="C:/msys64/mingw64/bin/mingw32-make.exe" \
  ..
```

### 2. Build (Incremental)

```bash
# From PowerShell (required — sets MSYS2 bins on PATH so gcc can find cc1/as/ld)
PATH=/c/msys64/mingw64/bin:/c/msys64/usr/bin:$PATH && /c/msys64/mingw64/bin/mingw32-make.exe
```

> **Note:** When Claude Code runs from PowerShell, MSYS2 tools are not on PATH.
> The `PATH=...` prefix is required or gcc fails silently.

Output: `pokered.exe`

### 3. Copy SDL2.dll

**CRITICAL:** Game will not run without this.

```bash
cp /c/msys64/mingw64/bin/SDL2.dll /c/Users/Anthony/pokered/build/
```

### 4. Launch Game

```bash
cd C:/Users/Anthony/pokered
bash tools/launch_game.sh
```

Sets correct CWD for save files and CLI tools.

## Common Issues & Solutions

### "cmake.exe not found"
- Make sure CMake is installed and in PATH
- Check: `cmake.exe --version`

### "gcc.exe not found"
- MinGW not in PATH. Use full paths as shown above.
- Do NOT try to use MSVC or NMake — this project requires MinGW/gcc

### "Cannot open pokered.exe — SDL2.dll not found"
- You forgot step 3. Copy SDL2.dll:
  ```bash
  cp /c/msys64/mingw64/bin/SDL2.dll /c/Users/Anthony/pokered/build/
  ```

### Build fails with "CMakeLists.txt:2: CMAKE_C_COMPILER not set"
- You deleted the `build/` directory or CMake lost configuration
- Full rebuild needed (see below)

### "lingking error" or strange compilation failures after code edits
- Clean rebuild (see below)

## Clean Rebuild (When Things Break)

```bash
cd C:/Users/Anthony/pokered
rm -rf build
mkdir build
cd build

/c/msys64/mingw64/bin/cmake.exe -G "MinGW Makefiles" \
  -DCMAKE_C_COMPILER="C:/msys64/mingw64/bin/gcc.exe" \
  -DCMAKE_MAKE_PROGRAM="C:/msys64/mingw64/bin/mingw32-make.exe" \
  ..

/c/msys64/mingw64/bin/mingw32-make.exe

cp /c/msys64/mingw64/bin/SDL2.dll .
```

Then: `bash ../tools/launch_game.sh`

## Key Invariants

1. **CMake is required** — This project uses CMake to generate Makefiles. You cannot just run `make` without it.

2. **MinGW only** — Do NOT use MSVC, NMake, or other Windows compilers. The project is configured for MinGW/gcc.

3. **Always use full paths** — Paths in CMake must be Windows-style (`C:/msys64/...`), not Unix-style (`/c/msys64/...`).

4. **SDL2.dll is separate** — MinGW build does NOT automatically copy SDL2.dll. You must copy it manually after building.

5. **CWD matters for launch** — Always use `bash tools/launch_game.sh` to launch the game, NOT `./pokered.exe` directly. The script sets the working directory so save files and CLI tools work correctly.
